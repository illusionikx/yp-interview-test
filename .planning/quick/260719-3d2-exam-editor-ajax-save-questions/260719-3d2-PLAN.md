<objective>
On the lecturer exam editor (Questions tab), a per-question Save currently does a full-page POST + redirect. Progressively enhance the per-question Save so that — on an exam with no attempts — it submits via the app's existing `window.axios` (X-Requested-With auto-set) and the page does NOT reload: the Alpine `dirty` flag clears, a lightweight "Saved" indicator shows, and a validation failure surfaces field errors inline (422) without a reload.

The change mirrors the EXISTING AJAX precedent already in this file's neighbourhood — `QuestionReorderController` returns `response()->noContent()` when `$request->ajax()` and a redirect otherwise; `questionReorder()` posts with `window.axios`. We add the identical 204-on-ajax branch to `ExamQuestionController::update()` and the identical axios submit to the per-question form.

Purpose: eliminate the jarring full-page refresh in the dominant authoring workflow (editing questions on a draft / no-attempt exam), which is exactly where the reload is annoying.

Output: an axios-enhanced per-question Save with inline errors + saved indicator, a 204-on-ajax controller branch, and a focused response-contract test. Full suite stays green.

Scope boundary (deliberate, do not expand):
- The attempted-exam Save path (warn-and-void) is LEFT EXACTLY AS TODAY: submit → confirm modal → native `$refs.questionForm.submit()` → server voids attempts → redirect + flash. Native `.submit()` does not carry X-Requested-With, so it hits the redirect branch. This preserves warn-and-void byte-for-byte AND guarantees the Submissions panel / delete-question modal / reset modal / every question form re-render truthfully after a permanent void — re-deriving all of that client-side would be over-engineering for the rare destructive path. AJAX enhances the common no-attempt path only.
- `store()` / `quickStore()` are untouched. The `_form` partial is currently only rendered in EDIT mode (per question in `show.blade.php`); its create/store instance is dormant (`questions/edit.blade.php` redirects — see STATE 12-02). No store() contract change.
</objective>
