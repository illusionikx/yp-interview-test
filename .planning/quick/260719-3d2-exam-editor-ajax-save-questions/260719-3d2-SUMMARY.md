# Quick Task 260719-3d2 — Exam-editor AJAX per-question save

## Goal
On the lecturer exam editor, per-question **Save** did a full-page refresh
(POST + redirect). Make it submit via AJAX so the page no longer reloads, while
preserving server validation, the warn-and-void behaviour on attempted exams,
and the existing Alpine dirty/discard UX. No new dependencies.

## What changed
- **`app/Http/Controllers/Lecturer/ExamQuestionController.php`** — `update()`
  now returns `response()->noContent()` (204) when `$request->ajax()`, mirroring
  the existing `QuestionReorderController` precedent. The native (non-ajax) path
  is byte-for-byte unchanged: same redirect, same warn-and-void flash.
- **`resources/views/lecturer/exams/questions/_form.blade.php`** — the per-question
  form is intercepted with `@submit.prevent`. On an exam with **no attempts** it
  posts `new FormData(form)` through the app's global `window.axios` (which already
  sends `X-Requested-With` + CSRF), clears the Alpine `dirty` flag, and shows a
  transient inline "Saved" indicator. A 422 populates an inline per-field error
  summary (first message per field; `options.*` collapsed) with no reload. Any
  unexpected error falls back to a native `form.submit()`.
- **`tests/Feature/Lecturer/ExamQuestionAjaxSaveTest.php`** (new) — pins the
  three-way response contract: 204 on ajax, redirect on non-ajax, 422 JSON on
  invalid ajax.

## Design decisions
- **Warn-and-void path left unchanged.** When the exam already has attempts, the
  confirm modal → native `$refs.questionForm.submit()` → redirect flow is
  untouched. A native `.submit()` carries no `X-Requested-With`, so it hits the
  redirect branch and a full page load correctly re-renders the Submissions panel
  and delete/reset modals after a permanent void. AJAX enhances only the common
  no-attempt authoring path.
- **Non-JS fallback preserved.** The `@submit.prevent` intercept is Alpine-only;
  with JS disabled the `<form>` posts natively to the same endpoint → redirect.
- **CSRF** satisfied by the form's existing `@csrf` token carried in `FormData`
  plus the app's configured axios headers. No manual header assembly, no new deps.
