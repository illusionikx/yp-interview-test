<objective>
Disarm the factory trap across the test suite — as a **behavior-neutral, suite-stays-green** pass,
before plan 06 makes it fatal.

Purpose: `ExamFactory:23` and `SectionFactory:21` EACH independently call `Subject::factory()`, so
`Exam::factory()` + `Section::factory()` land in **different subjects**. The `exam_section` pivot
masks this completely today — the fixture explicitly attaches the exam to the section, so visibility
works regardless of subject. Once plan 06 makes visibility subject-derived, every one of these
fixtures silently breaks.

Splitting this pin from plan 06's pivot drop is deliberate: **pinning `subject_id` is inert while the
pivot is still authoritative.** The exam is visible because of the pivot row either way. So this whole
plan is verifiable by a single fact — nothing changed. That gives plan 06 a clean, already-correct
foundation instead of forcing it to do semantic fixture work and a schema break in one breath.

Output: 16 test files whose fixtures pin subject explicitly. Zero assertion changes. Zero production
code.
