<objective>
Ship EDT-04's frontend: make the editor reachable on a published exam, and intercept the save with the
warning that makes destruction non-silent.

Purpose: plan 08 relaxed the gates server-side, but the affordances are still hidden behind
`@unless ($exam->is_published)` wrappers — so today the backend allows an edit no one can click. This
plan closes that gap, and adds the interception that is EDT-04's actual requirement: *"if any student
has attempted previously, a warning pops up first."*

The lecturer's mental model walking in is "I'm saving a form", not "I'm about to delete data". The copy
must make that translation explicit rather than assume it — which is why the modal's title names both
halves of the act.

Output: editor always reachable, three forms intercepted from one copy source, `save-exam-changes`
modal live.
</objective>
