<objective>
Delete the manual exam-assignment feature outright (D-1 / FIX-03), and make the draft↔published
toggle work in both directions (CLS-06).

Purpose: CLS-05 states that visibility **is** subject enrollment, which makes a manual per-class
assignment step dead weight. **FIX-03 is satisfied BY REMOVAL** — "Update assignment gives no visible
feedback" cannot exist once the button that caused it is gone. Do NOT fabricate a toaster for a
deleted screen.

`exam_section` pivot table itself is still present. That split is deliberate and keeps the suite
green: the pivot is only reachable from the model relation after this plan, and plan 06 removes the
relation and the table together.

Output: an exam detail page with no assignment panel, a toggle that works post-attempt, and two
toasts that tell the lecturer what actually happened.
</objective>

**Production change — `ExamController::unpublish()`:**
  - Delete the `if ($exam->attempts()->exists()) { return back()->with('status', 'Cannot unpublish an exam that students have already attempted.'); }` guard block entirely. The method becomes a bare
    `$exam->update(['is_published' => false]);` plus its redirect — identical in shape to `publish()`.
  - Toggling touches no attempt data in either direction; neither method references `attempts` once the
    guard is gone. It only flips `is_published`. No confirm-modal is needed — it is non-destructive, so
    it stays the single-click action it already is.

**Toast copy (success criterion 5), verbatim from the UI-SPEC Copywriting Contract:**
  - `publish()`: `back()->with('status', __('Exam published. Students can now see and start it.'))`
    (replacing the current bare `'Exam published.'`)
  - `unpublish()`: `back()->with('status', __('Exam moved back to draft. Students can no longer start it, but existing attempts are unaffected.'))`
    The second clause is load-bearing, not padding: it is the one fact a lecturer needs in order to use
    this control freely — that toggling is safe and does not touch attempt data. Flash key is `status`
    (`<x-toast>` reads `status`/`error`; `session('success')` has zero call sites — do not introduce it).

**Rewrite `unpublish()`'s doc comment.** The existing comment states the exam "stays published and
immutable" once attempted, justified by Phase-5 review finding HIGH-02: unpublishing would re-open the
draft-only edit gate, letting a lecturer change correct answers after grading and desync computed
scores. Record that HIGH-02's protection is **superseded, not abandoned** — D-4/D-6 retire the
draft-only gate outright, and EDT-04's warn-and-void flow (plan 08) replaces it with a stronger,
explicit guarantee: editing an attempted exam destroys its attempts, after a warning. The old lock was
"you may not touch it"; the new protection is "touching it costs you the attempts, and you will be told
first." Reference CLS-06 and D-4.
