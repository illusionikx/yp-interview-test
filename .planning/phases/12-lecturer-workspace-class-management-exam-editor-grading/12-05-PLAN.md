<objective>
Add authoring-time reordering to the exam editor's Questions tab: move-up / move-down buttons beside each question's number (EDT-05) and beside each of a question's options, plus a single per-question "shuffle" button that randomizes that question's option arrangement once and stores it (EDT-03). All operations mutate the existing `questions.position` / `options.position` columns via a per-swap POST (no migration — the columns and fillable already exist) and every render is `orderBy('position')`. Per Decision #8 these are move-up/down buttons, never drag-and-drop; per Decision #2 the shuffle is authoring-time only — it rearranges the stored order once when clicked, and there is no per-student runtime randomization (TAK-12 is the student-side mirror in Phase 13).

Reordering is display-order only: `options.is_correct` is stored per option, so swapping positions never changes grading — therefore reorder/shuffle are NON-destructive and, unlike an edit, do NOT trigger the EDT-04 warn-and-void. This is deliberate and encoded in the tests.

Purpose: Let a lecturer arrange questions and answers deterministically, once, at authoring time — Dusk-testable, zero new dependencies.
Output: three reorder routes + a dedicated `QuestionReorderController`; default position ordering on the Exam/Question relationships; move/shuffle buttons on the Questions tab; QuestionReorderTest.
</objective>

  <behavior>
    - `moveQuestion(exam, question)` with direction up|down swaps the question's `position` with the adjacent sibling's; at the top with up (or bottom with down) it is a no-op.
    - `moveOption(exam, question, option)` with direction up|down swaps the option's `position` with the adjacent sibling within the same question; boundary is a no-op.
    - `shuffleOptions(exam, question)` writes a random permutation of `0..n-1` across the question's options' `position` values in one transaction.
    - Every action 404s on a mismatched exam/question (or question/option) pair; every action redirects back to the editor Questions tab; none of them deletes or voids any attempt.
    - After any action, `Exam::questions` and `Question::options` read back in ascending `position`.
  </behavior>
  <action>
    Add default ordering so every consumer renders correctly regardless of call site: in `Exam::questions()` return `$this->hasMany(Question::class)->orderBy('position')`, and in `Question::options()` return `$this->hasMany(Option::class)->orderBy('position')`. (Aggregates like `->max('position')` ignore orderBy, so ExamQuestionController's position-assignment is unaffected.)

    Create `app/Http/Controllers/Lecturer/QuestionReorderController.php` with three actions, each opening with the nested-binding integrity check reused from ExamQuestionController (`abort_unless($question->exam_id === $exam->id, 404)`, and for options `abort_unless($option->question_id === $question->id, 404)`):
    - `moveQuestion(Request, Exam $exam, Question $question)`: validate `direction in:up,down`. In a transaction, find the adjacent sibling among `$exam->questions` by position (up = the greatest position less than this one; down = the least position greater than this one). If none (boundary), redirect back unchanged. Otherwise swap the two `position` values and save both.
    - `moveOption(Request, Exam $exam, Question $question, Option $option)`: same swap among `$question->options`.
    - `shuffleOptions(Exam $exam, Question $question)`: load the question's options, produce a shuffled sequence of the integers `0..count-1`, assign them to the options and save, all in one transaction. This is the authoring-time one-shot (Decision #2) — do NOT randomize at any read/take path.
    - Each action returns `redirect()->route('lecturer.exams.show', $exam) . ?tab=questions` (redirect back to the Questions tab) with a `->with('status', ...)` toast. NONE of them calls AttemptVoider or deletes anything — reordering is non-destructive (is_correct is untouched), so no EDT-04 warning applies.

    Register three routes in routes/lecturer.php inside the existing `role:lecturer` group, beside the current `exams/{exam}/questions/...` routes, using PATCH:
    - `PATCH exams/{exam}/questions/{question}/move` → `[QuestionReorderController::class, 'moveQuestion']` name `exams.questions.move`
    - `PATCH exams/{exam}/questions/{question}/options/{option}/move` → `[QuestionReorderController::class, 'moveOption']` name `exams.questions.options.move`
    - `PATCH exams/{exam}/questions/{question}/options/shuffle` → `[QuestionReorderController::class, 'shuffleOptions']` name `exams.questions.options.shuffle`
    Import the controller at the top.
  </action>
