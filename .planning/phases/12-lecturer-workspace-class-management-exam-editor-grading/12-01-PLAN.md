<objective>
Deliver the per-subject Lecturer Workspace hub shell and its Classes tab (CLS-01, CLS-02) plus the class form's location field (CLS-03). One new subject-scoped, ownership-gated page hosts two Alpine tabs — Classes (default) and Exams (stubbed here, filled by 12-04) — reached from the Phase 11 lecturer subject list. Class CRUD reuses the existing `SectionController` and Store/UpdateSectionRequest (class = Section, copy-only relabel per Decision #3); this plan adds the one genuinely missing field (`location`) and re-presents the section list per subject with a total/max progress bar.

Purpose: Give a lecturer one place to run a subject's classes, and close the door on the currently-unscoped class listing by making the canonical listing subject-scoped and ownership-gated.
Output: `lecturer.subjects.manage` route + controller + two-tab view + Classes partial + Exams stub partial; a nullable `location` column and its form field; retargeted class-CRUD redirects and the home "Manage classes" link.
</objective>
