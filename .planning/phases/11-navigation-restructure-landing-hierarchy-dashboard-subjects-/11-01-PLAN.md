  <action>
    Create three Blade components following the existing @props + $attributes->merge convention.
    (1) x-back-button: declares a required `href` prop; renders an anchor styled as a button
    (inline-flex items-center, rounded-base or rounded-lg, border, subtle background) containing
    a small left-arrow glyph followed by the slot. The SLOT carries the destination-naming text
    (e.g. "Back to classes"); the component must NOT hardcode any destination word. This is the
    UX-04 primitive: a back affordance that is a button whose text names where it leads.
    (2) x-welcome-banner: optional `name` prop defaulting to auth()->user()->name; renders a
    full-width banner using a brand gradient (bg-gradient-to-r from the brand tokens, e.g.
    from-brand to-brand-strong) with white heading text "Welcome back, {name}". Purely
    decorative, no data dependency — this is DASH-02.
    (3) x-dashboard-card: declares `label` and `value` props and an optional `progressCurrent`
    /`progressMax` pair; renders a stat card on bg-neutral-primary-soft with border-default and
    rounded-base, showing the label, the large value, and — when both progress props are
    provided — a proportional progress bar (width computed from current/max, clamped 0..100).
    This card is consumed by 11-02 and 11-03; keep it presentational only (no queries inside).
  </action>
