  </read_first>
  <behavior>
    - subjectsThisSemester: the count of DISTINCT subjects the student is Enrolled in whose section
      falls in the CURRENT semester (composite ordinal equality), excluding past/future.
    - examsAvailable: the count of exams visible to the student (Exam::visibleTo) that are within
      their availability window and that the student has not yet attempted. Already-attempted or
      out-of-window exams are excluded. Bounded COUNT, no PHP loop.
    - groupedSubjects: the student's Enrolled subjects, each with lecturer name(s) and the section's
      (year, semester), grouped into semester buckets and split into current+future vs past.
      A student with no enrollments gets zero cards and an empty list, not an error.
  </behavior>
  <action>
    Create app/Http/Controllers/Student/HomeController.php with index(Request). Resolve $student =
    $request->user(). Compute the two bounded aggregates (COUNT only):
      - subjectsThisSemester: count DISTINCT subject_id over Sections whose composite ordinal equals
        the current ordinal AND which have an Enrolled enrollment for this student. Express the
        ordinal match in SQL with whereRaw('(year * 2 + (2 - semester)) = ?', [Semester::current()->ordinal()])
        and whereHas('enrollments', fn ($q) => $q->where('user_id', $student->id)->where('status', EnrollmentStatus::Enrolled))
        then ->distinct()->count('subject_id'). Comment that the raw expression mirrors Semester::ordinal().
      - examsAvailable: Exam::visibleTo($student)
          ->whereDoesntHave('attempts', fn ($q) => $q->where('user_id', $student->id))
          ->where(fn ($q) => $q->whereNull('available_from')->orWhere('available_from', '<=', now()))
          ->where(fn ($q) => $q->whereNull('available_until')->orWhere('available_until', '>', now()))
          ->count().
    Build the NEW enrolled-subjects list (NOT the SubjectBrowse catalog): load the student's
    Enrolled enrollments with the section, subject and subject.lecturers eager-loaded in ONE query
    (Enrollment::where('user_id', $student->id)->where('status', EnrollmentStatus::Enrolled)
    ->with(['section.subject.lecturers'])->get()). Transform (iterating the already-loaded collection,
    which is not an N+1) into a structure grouped by the section's Semester label, each entry
    carrying subject code/name, joined lecturer name(s) (or "Unassigned"), the Semester value object,
    and whether that Semester isPast(). Order groups by ordinal descending. Pass subjectsThisSemester,
    examsAvailable, and the grouped structure (split into current-or-future groups and past groups) to
    view('student.home', ...). In routes/student.php replace the home closure with
    Route::get('/', [\App\Http\Controllers\Student\HomeController::class, 'index'])->name('home');
    (keep name 'home' and the role:student group unchanged).
  </action>
  <acceptance_criteria>
    - HomeController references Semester::current(), Exam::visibleTo, and whereDoesntHave('attempts', ...).
    - subjectsThisSemester uses the literal pattern `(year * 2 + (2 - semester))`.
    - The enrolled-subjects query eager-loads section.subject.lecturers (grep confirms
      'section.subject.lecturers'); no per-row query: `grep -nE "->lecturers\(\)->get|foreach.*->sections" app/Http/Controllers/Student/HomeController.php` returns nothing problematic.
    - routes/student.php home route points at Student\HomeController@index.
    - php artisan test --filter=RoleRedirectTest passes.
  </acceptance_criteria>
  <verify>
    <automated>php artisan test --filter=RoleRedirectTest</automated>
  </verify>
  <done>student.home resolves to a controller giving two scoped aggregates + the new enrolled-by-semester structure.</done>
