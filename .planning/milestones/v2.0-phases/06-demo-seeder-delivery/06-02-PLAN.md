## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| CLI seeder → MySQL `yp-student-exam` | The seeder writes directly via Eloquent in a dev/CLI context; it exposes no HTTP surface and is not reachable by an end user. |
| seeded accounts → (later) public GitHub repo | Fixed demo passwords are written into the DB and documented publicly — a deliberate, scoped demo convenience. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-06-01 | Information Disclosure | Fixed weak demo password (`password`) on seeded accounts | accept | Deliberate, disclosed demo convenience; passwords are hashed via the User model's `hashed` cast (never stored plaintext). README (06-03) frames them as demo/evaluation-only, not a production posture. |
| T-06-02 | Elevation of Privilege | Seeder mass-assigns User.role / classroom_id | mitigate | This is the documented server-controlled carve-out (User model docblock, Pitfall 7); confined to the seeder — no request-sourced input reaches these attributes. No public path is created. |
| T-06-03 | Tampering | Idempotency / re-seed clobbering reviewer state | mitigate | firstOrCreate (not updateOrCreate) on natural keys + `wasRecentlyCreated`-guarded children + `sync()` pivots — a re-run never duplicates rows and never overwrites a manually-graded open-text answer. |
| T-06-SC | Tampering | composer/npm installs | accept | No packages installed this phase (06-RESEARCH.md Package Legitimacy Audit: none). No `[ASSUMED]`/`[SUS]` package → no legitimacy checkpoint required. |
