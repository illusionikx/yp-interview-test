<objective>
Deliver the second vertical slice: answers save as the student enters them. A per-answer AJAX POST persists each MCQ selection or open-text response through the same server deadline gate as submit, and the take page shows a per-question save status. Because 04-02 already rehydrates saved answers on show(), autosave makes refresh/disconnect fully non-destructive.

Purpose: TAK-03 (incremental autosave surviving reload) and TAK-02's deadline gate on the write path (D-04), via one idempotent updateOrCreate endpoint (D-06) guarded by AnswerRequest input validation (V5, CWE-915).
Output: AnswerRequest, AttemptController@answer, the attempts.answer route, and the Alpine autosave wiring + status tags on the take page.

Slice capability gained: after this plan a student can answer questions and every answer is saved immediately; reloading mid-exam preserves their work; late writes are refused server-side.
</objective>
