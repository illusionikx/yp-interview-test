<objective>
Deliver the first vertical slice: a student clicks "Start Exam" (or "Resume Exam"), a single server-timed attempt is created or resumed, and they land on the take page seeing all questions — with the server as the sole timing authority and the answer key never leaving the server. An expired attempt auto-finalizes on visit and redirects to a plain confirmation.

Purpose: Stand up the server-authoritative core (D-01 single-attempt, D-03 server deadline, D-04/D-05 lazy finalize, D-07 no-leak, D-08 policy, D-10 seam). This is the substance of the phase's correctness guarantees; 04-03 (autosave) and 04-04 (submit/live-countdown) layer onto the take page and controller this plan creates.
Output: Attempt timer methods, AttemptPolicy, AttemptController@store/show/submitted, three routes, the take page + submitted views, and the activated Start seam.

Slice capability gained: after this plan a student can start/resume an assigned exam and read the whole paper; a lapsed attempt closes itself on next touch. (Autosave and live countdown/submit arrive in 04-03/04-04.)
</objective>
