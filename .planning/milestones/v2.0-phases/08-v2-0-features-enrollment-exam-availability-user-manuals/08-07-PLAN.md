<objective>
Enforce the exam availability window at attempt start, and turn the existing exam landing page
into the AVL-02 pre-start details page.

Purpose: this is the enforcement half of exam availability. Its whole design risk is *where*
the check goes. Availability is a narrow additive check at exactly one call site — never part
of the shared visibility predicate, never part of the attempt policy — because a closed window
must refuse a NEW attempt while leaving an already-started one fully resumable. Getting this
wrong reintroduces the AVL-04 bug 08-03 just fixed, from a different direction.

Output: one guarded branch in AttemptController@store, an enriched pre-start page, an
availability pill on the exam list.
</objective>

<binding_decisions>
- **Availability must NEVER enter `Exam::scopeVisibleTo()`.** It is a narrow additive check
  (`Exam::isAvailableNow()`, built in 08-01) applied at exactly ONE call site:
  `AttemptController@store`, new-attempt branch only. An already-started attempt stays resumable
  regardless of window state.
- **Post-start attempt access is ownership-gated, not visibility-gated**; the
  enrollment/availability check applies **at start only** (REQUIREMENTS.md #7). 08-03 already
  implemented this in `AttemptPolicy` — do not revisit it.
- Window boundaries are half-open `[available_from, available_until)`; a null bound is unbounded
  (REQUIREMENTS.md #6).
- The pre-start page **reuses the existing `student.exams.show` route/view** — no new route.
- Pre-start actions are **Proceed** (primary) and **Back** (secondary, returns to
  `student.exams.index`).
</binding_decisions>
