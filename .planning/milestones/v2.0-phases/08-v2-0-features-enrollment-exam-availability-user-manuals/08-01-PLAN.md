## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| student browser -> enrollments table | Untrusted student input will (from 08-04 onward) drive writes to a Pivot model whose every column is mass-assignable. This plan sets the invariant that protects it. |
| lecturer browser -> exams.available_from/available_until | Untrusted datetime input reaching two new columns that gate attempt starts (08-07). |
| stored window columns -> authorization decision | `isAvailableNow()` is read by the AVL-03 gate; a wrong boundary here is an access-control bug, not a cosmetic one. |
