<objective>
## Phase Goal (MVP slice)

**As a** lecturer, **I want to** assign an exam to one or more classrooms, **so that** only students in those classrooms can ever be given access to it once it is published.

Deliver the lecturer-side write half of Phase 3 (ASN-01): a dedicated single-purpose `Lecturer\ExamAssignmentController@update` that `sync()`s the `exam_classroom` pivot from a validated `classroom_ids` array, plus the "Assign to classes" multi-select panel on the existing lecturer exam show page. Assignment is editable regardless of publish state (D-01) — student visibility is gated separately by `is_published` in the student slice (03-03).

Purpose: Make the exam→classroom relationship a lecturer can set, the precondition for class-scoped student visibility.
Output: `AssignExamRequest`, `ExamAssignmentController`, one PUT route, and the show-page assign panel — turning `tests/Feature/Lecturer/ExamAssignmentTest.php` from RED to GREEN.
</objective>

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| browser → lecturer assignment endpoint | A form POST/PUT carrying `classroom_ids[]` crosses from the (authenticated lecturer) client into the server; the ids are untrusted input. |
| role boundary (student ↔ lecturer) | A student may attempt to reach the lecturer-only assignment route directly. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-3-03 | Elevation of Privilege | `ExamAssignmentController@update` reached by a non-lecturer | mitigate | Route sits inside the existing `role:lecturer` group in `routes/lecturer.php`; a student is 403 before the controller runs (asserted by `test_a_student_is_forbidden_from_assigning_an_exam`). |
| T-3-05 | Tampering | forged/invalid `classroom_ids` payload | mitigate | `AssignExamRequest` validates `classroom_ids.*` as `integer, distinct, exists:classrooms,id` before `sync()`; any non-existent id is rejected (asserted by `test_assignment_rejects_a_nonexistent_classroom_id`). No ownership scoping needed — `Classroom` has no per-lecturer owner column (confirmed in 03-RESEARCH.md Security Domain). |
| T-3-06 | Information Disclosure | assigning a draft exam must NOT expose it to students | accept-here / mitigate-in-03-03 | This controller intentionally allows draft assignment (D-01); the visibility gate is `is_published` inside `scopeVisibleTo`/`ExamPolicy` (03-03), not here. Disposition noted so the executor does not add an `is_published` guard to this endpoint. |
| T-3-SC | Tampering | npm/composer installs | accept | No new packages (D-06); `make:controller`/`make:request` scaffold plain PHP against framework code already in `composer.lock`. |
