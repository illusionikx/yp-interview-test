<objective>
## Phase Goal (MVP slice)

**As a** student, **I want to** see and open only the published exams assigned to my own classroom, **so that** I can find my exams and can never reach one that is not mine — even by guessing its URL.

Deliver the RBAC-05 core: one shared `Exam::scopeVisibleTo($query, $user)` predicate ("published AND assigned to this student's classroom", with an explicit null-classroom guard) used by BOTH the student index list AND `ExamPolicy::takeable()` so the list and the direct-access gate can never diverge (the "hidden-but-reachable" bug class this phase exists to prevent). The student `show` route is protected by `$this->authorize('takeable', $exam)` → 403, not by route-model binding alone. The landing page is read-only with a disabled Phase-4 "Start" seam.

Purpose: Make class-scoped exam access provably enforced server-side (no IDOR), establishing the ExamPolicy authorization convention Phase 4/5 will copy for attempts/results.
Output: `Exam::scopeVisibleTo`, `ExamPolicy`, `Student\ExamController`, student routes + views, and the `AuthorizesRequests` trait on the base controller — turning `ExamIndexTest` and `ExamAccessTest` from RED to GREEN.
</objective>

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| browser → student exam show route | A student (or lecturer) can type/guess `/student/exams/{id}` for ANY exam id; the id is untrusted. Route-model binding proves the row exists but says nothing about authorization. |
| role boundary (lecturer → student area) | A lecturer may attempt to reach the student-only exam routes directly. |
| index list ↔ direct-access gate | Two code paths ("what the list shows" vs "what a direct URL allows") that must agree; divergence is itself the vulnerability. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-3-01 | Information Disclosure / Elevation | `Student\ExamController@show` (IDOR — guessed URL to an unassigned/unpublished exam) | mitigate | `$this->authorize('takeable', $exam)` runs before render → `AuthorizationException` → 403; never relies on route-model binding or list-omission alone (asserted by `ExamAccessTest` cases 2–4). |
| T-3-02 | Elevation of Privilege | index filter and policy predicate drifting apart ("hidden but reachable") | mitigate | ONE `Exam::scopeVisibleTo()` called by name from both `@index` and `ExamPolicy::takeable()` — there is no second query to drift (research Pitfall 1). |
| T-3-04 | Information Disclosure | student with `classroom_id = null` accidentally matching exams | mitigate | Explicit `->when($user->classroom_id, ..., fn => whereRaw('0 = 1'))` branch → null classroom yields zero rows for both list and gate (asserted by the null-classroom index + direct-access cases). |
| T-3-06 | Information Disclosure | a draft exam pre-assigned to a classroom leaking before publish | mitigate | `scopeVisibleTo()` always ANDs `is_published = true` with the classroom `whereHas` (asserted by `ExamIndexTest` case 2 + `ExamAccessTest` case 3). |
| T-3-07 | Information Disclosure | student landing page leaking questions / correct options | mitigate | `show()` uses `loadCount('questions')` (a count only), the view renders no question bodies/options; taking the exam is Phase 4. Start affordance is a disabled seam. |
| T-3-SC | Tampering | npm/composer installs | accept | No new packages (D-06); `make:policy`/`make:controller` scaffold plain PHP against framework code already resolved in `composer.lock`. |
</threat_model>
