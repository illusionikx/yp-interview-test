<objective>
Wave 0 for Phase 5. Establish the executable acceptance contract for grading & results BEFORE any production code exists: the Answer/Attempt factory states needed to build the auto-grade edge-case matrix cheaply, and the four RED feature test classes that pin down GRD-01..GRD-05. No production grading code is written in this plan — the tests fail RED and become the fixed contract Waves 2-4 must satisfy (matching the 03-01 / 04-01 Wave-0 precedent).

## Phase Goal (MVP user stories)

- **As a** student, **I want to** see my accurate exam score and a per-question breakdown once grading is complete, **so that** I know how I performed — but never a partial score before grading is done, and never the answer key.
- **As a** lecturer, **I want** multiple-choice answers auto-graded on submission and a place to grade open-text answers and review results per exam and per student, **so that** every attempt ends in an accurate, reviewable score.

Purpose: Lock the grading correctness matrix (unanswered/wrong/correct MCQ, open-text completeness gate), the ownership-only result policy, and the no-answer-key rule as failing tests first, so downstream implementation is measured against them.
Output: 2 factory files extended + 4 RED feature test classes.
</objective>
