## Phase Goal (User Story)

**As a** lecturer, **I want to** build the full teaching content model — classrooms, subjects, class-subject links, student rosters, and complete exams — **so that** everything is authored and correct before any of it is exposed to a student.

*(Derived from the Phase 2 `**Goal:**` line and success criteria in `.planning/ROADMAP.md` — not invented scope.)*

<objective>
Deliver the Nyquist "Wave 0" test infrastructure every subsequent Phase-2 feature plan depends on: four brand-new Eloquent factories (`SubjectFactory`, `ExamFactory`, `QuestionFactory`, `OptionFactory`), convenience `lecturer()`/`student()` states on the existing `UserFactory`, and the `tests/Feature/Lecturer/` directory. Phase 1 built the models and migrations; it did NOT build these factories (only `ClassroomFactory` and `UserFactory` exist).

Purpose: Feature tests for CLS-01..04 and EXM-01..06 (per `02-VALIDATION.md`'s test map) cannot run until these fixtures exist. Scheduling them first unblocks every later wave.
Output: 4 new factories, 1 modified factory, the `tests/Feature/Lecturer/` directory, and a smoke test proving the factories persist valid models.

This plan is test infrastructure only — it produces no lecturer-facing capability. It exists because `02-RESEARCH.md` §Wave 0 Gaps and `02-VALIDATION.md` §Wave 0 Requirements both mandate these factories before the first dependent feature test. Requirements listed above are the ones whose feature tests most directly require the new `mcq()`/`open()`/`published()` factory states; those requirements are owned and implemented by plans 02-04 and 02-05.
</objective>
