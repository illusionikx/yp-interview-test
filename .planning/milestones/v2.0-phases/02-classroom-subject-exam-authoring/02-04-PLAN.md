## Phase Goal (User Story)

**As a** lecturer, **I want to** build the full teaching content model — classrooms, subjects, class-subject links, student rosters, and complete exams — **so that** everything is authored and correct before any of it is exposed to a student.

*(Derived from the Phase 2 `**Goal:**` line in `.planning/ROADMAP.md`.)*

**This slice:** *As a lecturer, I want to create an exam under a subject with a title and time limit, edit/delete it while it's a draft, and publish it, so that it becomes eligible for assignment.*

<objective>
Deliver exam authoring shell and lifecycle (EXM-01, EXM-05, EXM-06; D-05, D-06): create an exam belonging to a subject with `title`, optional `description`, and `duration_minutes`; land on the exam's own show page (which will host question authoring in 02-05); edit/delete only while unpublished; and publish/unpublish via two explicit named routes. `created_by` is stamped from the authenticated lecturer server-side.

This slice implements the exam-level half of EXM-05 (the exam's own edit/delete gate). The question-level half of EXM-05 (blocking question mutations on a published exam) is implemented and tested in 02-06 via `ExamPublishedEditGateTest`; together they satisfy EXM-05.

Purpose: An exam must exist and be publishable before questions can be authored (02-05/02-06) and before Phase 3 assigns it to classrooms.
Output: `Lecturer\ExamController` (CRUD + publish/unpublish), Store/UpdateExamRequest (update carries the draft-only gate), exam Blade views incl. `show`, routes, and two feature tests.
</objective>

|----------|-------------|
| browser → exam routes | Exam form input + publish/unpublish actions cross into `ExamController` behind Phase-1 `auth`+`verified`+`role:lecturer`. |
| published-state boundary | A published exam becomes immutable content; the mutation gate is the boundary. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-RBAC | Elevation of Privilege | exams.* + publish/unpublish routes | mitigate | Inside the unchanged Phase-1 `role:lecturer` group; ExamControllerTest + ExamPublishTest assert student 403. |
| T-02-MA | Tampering / EoP | StoreExamRequest / ExamController@store | mitigate | `created_by` is merged from `auth()->id()` server-side; a forged `created_by` in the request body is never mass-assigned (T-01-02-MA); tested explicitly. |
| T-02-PUB | Tampering | UpdateExamRequest.authorize() + destroy guard | mitigate | Draft-only mutation gate: `authorize()` returns `! route('exam')->is_published`; destroy guards `abort_if($exam->is_published,403)`. | T-02-INPUT | Tampering | exam validation | mitigate | `subject_id` exists:subjects; `title` required; `duration_minutes` integer|min:1 — rejects malformed exam data server-side (V5). |
