# Phase 7 — UI Design Contract

> Visual and interaction contract for frontend phases. Generated by gsd-ui-researcher, verified by gsd-ui-checker.

**Scope of this contract:** the Flowbite admin shell (top navbar + dropdowns, card content, dark-mode toggle) applied app-wide, the semantic status-pill system (UI-01), lecturer-facing section CRUD (capacity, enrollment open/close dates), and subject→lecturer assignment. FIX-01 (answered-count) and the schema/`scopeVisibleTo()` rewrite are non-visual and carry no new UI surface — no copy or layout changes beyond what's captured here for the existing submit-confirmation modal.

## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon-to-label gaps, pill internal padding |
| sm | 8px | Compact element spacing (nav-link internal padding, table cell vertical padding) |
| md | 16px | Default element spacing, card internal padding (mobile), form field gaps |
| lg | 24px | Card internal padding (desktop), section CRUD form group spacing |
| xl | 32px | Layout gaps between cards/sections on a page |
| 2xl | 48px | Major section breaks (e.g. between "Sections" card and "Assigned Lecturers" card on a subject page) |
| 3xl | 64px | Page-level top/bottom spacing on the main content area |

Exceptions:
- **44px minimum touch target** for icon-only controls (dark-mode toggle button, hamburger menu, dropdown-trigger carets) — overrides the 8-pt scale at this one point, per WCAG 2.5.5 (Target Size). Visual icon stays 20-24px; the clickable hit area (padding) is what reaches 44px.
- **Comfortable density** (07-CONTEXT.md, locked): cards use Flowbite's default `p-4` (mobile) / `p-6` (desktop) — i.e. `md`/`lg` tokens above, not a compact `sm`-padded variant.

## Typography

Exactly 4 sizes, exactly 2 weights (regular 400 + semibold 600) — no `font-medium`(500) usage in new Phase 7 markup, even though Figtree loads 500 for legacy Breeze components being replaced.

| Role | Size | Weight | Line Height |
|------|------|--------|-------------|
| Body | 16px (`text-base`) | 400 regular | 1.5 |
| Label | 14px (`text-sm`) | 600 semibold | 1.4 — used for nav-dropdown items, table column headers, status pills, form `<label>` text |
| Heading | 20px (`text-xl`) | 600 semibold | 1.2 — card/panel titles ("Sections", "Assigned Lecturers"), modal titles |
| Display | 28px (`text-3xl` rounded down from Tailwind's `text-3xl`=30px; use `text-3xl` class, treat as ~28-30px) | 600 semibold | 1.2 — page-level `$header` slot title (e.g. "Manage Sections") |

## Color

Flowbite default blue accent — **no custom Tailwind color config** (07-CONTEXT.md, locked). All values below are stock Tailwind/Flowbite palette classes.

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `bg-gray-50` light / `dark:bg-gray-900` dark | Page background, `<body>`/main content wrapper |
| Secondary (30%) | `bg-white` light / `dark:bg-gray-800` dark | Navbar, dropdown panels, cards, table surfaces, modals |
| Accent (10%) | `blue-600` light / `blue-500` dark (Flowbite default) | See "Accent reserved for" below |
| Destructive | `red-600` light / `red-500` dark | Destructive actions only (see below) |

Accent reserved for (explicit list — never "all interactive elements"):
- Primary CTA buttons: "Create Section", "Save Changes", "Assign Lecturer"
- Active/current nav-dropdown item indicator and link hover/focus state
- Text links (e.g. "View roster", breadcrumb current-page indicator)
- Focus rings (`focus:ring-blue-500`) on all interactive controls, including non-blue buttons
- Dark-mode toggle's active/pressed visual state

Destructive (`red-600`/`dark:red-500`) reserved for:
- "Delete Section" button and its confirmation-modal confirm button
- "Unassign Lecturer" button and its confirmation-modal confirm button
- (Not the Rejected/Closed status pills — those are a separate semantic system, below, not treated as "destructive interactive" color.)

### Status Pill Palette (semantic, locked per 07-CONTEXT.md — UI-01)

Distinct from the interactive accent/destructive colors above: these are informational badges, not buttons/links, and use Flowbite's badge component pattern (`rounded-full px-2.5 py-0.5 text-xs font-semibold`, sized down from the Label role for compact pill fit).

| State | Color | Applies to |
|-------|-------|-----------|
| Green | `bg-green-100 text-green-800` / `dark:bg-green-900 dark:text-green-300` | Enrolled, Published, Open |
| Gray | `bg-gray-100 text-gray-800` / `dark:bg-gray-700 dark:text-gray-300` | Draft, Withdrawn, "Opens …" |
| Red | `bg-red-100 text-red-800` / `dark:bg-red-900 dark:text-red-300` | Rejected, Closed |
| Amber | `bg-amber-100 text-amber-800` / `dark:bg-amber-900 dark:text-amber-300` | FULL |

Build this as one reusable Blade component (e.g. `x-status-pill status="published"`) mapping a status string to one of these four palettes — do not hand-roll pill markup per view.

## Phase Notes (shell-specific, for planner/executor)

- **Top navbar with dropdown navigation — NO sidebar.** This overrides the ROADMAP/REQUIREMENTS "sidebar navigation" wording per explicit user decision in 07-CONTEXT.md. Build with Flowbite's navbar + dropdown components, not a sidebar layout.
- Brand: plain text wordmark **"Exam Portal"** in the navbar — replaces `x-application-logo` in the nav (the SVG logo component itself is not deleted, just no longer used in the navbar slot).
- Role-based nav grouped into top-bar dropdowns: Lecturer → Subjects / Sections / Exams / Results. Student → Enroll / My Exams / Results.
- Content container: keep Breeze's existing `max-w-7xl mx-auto` centered width under the new top nav (conventional for a top-nav admin shell); card padding per the Spacing Scale exceptions above.
- Dark mode: `tailwind.config.js` → `darkMode: 'class'`. Toggle persists to `localStorage`, defaults to `prefers-color-scheme` when unset, and the `dark` class is applied via an inline `<head>` script **before** first paint (no flash of wrong theme). Implement the toggle as an Alpine component reusing the project's existing Alpine 3 usage pattern — no new JS framework.
- Reskin targets (existing views, no new IA): `resources/views/lecturer/{classrooms,subjects,exams,results}`, `resources/views/student/{exams,attempts,results}`, `dashboard.blade.php`, plus new views for section CRUD and subject→lecturer assignment.
- Section CRUD and subject→lecturer assignment page placement (nested under subject vs. standalone) is Claude's discretion per 07-CONTEXT.md — pick nested-under-subject unless a plan-time constraint argues otherwise, for IA consistency with the existing classroom→subject nesting pattern.
