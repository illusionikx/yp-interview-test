## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| npm registry → build pipeline | Flowbite is pulled into the Vite build (new third-party JS/CSS) |
| browser localStorage / OS preference → DOM | Dark-mode choice is read client-side pre-paint (client-only, no server round-trip) |
| status string → rendered class attribute | The status-pill maps caller-supplied status text to CSS classes |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-07-SC | Tampering | `npm install flowbite` | mitigate | flowbite verdict OK/Approved in 07-RESEARCH Package Legitimacy Audit (npm registry, 456k wk downloads, github.com/themesberg/flowbite, no postinstall script), re-verified via package-legitimacy + npm view; slopcheck on install; not [ASSUMED]/[SUS] so no blocking-human checkpoint required |
| T-07-01 | Information Disclosure / Tampering (XSS) | `x-status-pill` status prop | mitigate | Status string mapped through a fixed `match()` allowlist to static class strings; unknown status falls back to gray; label rendered as escaped Blade text (never `{!! !!}`, never interpolated into the class attribute) |
| T-07-02 | Information Disclosure | Dark-mode preference in localStorage | accept | Non-sensitive client-only UI preference; no server round-trip, no PII, no auth impact |
| T-07-SC-view | Tampering | Alpine `answeredCount` DOM state | accept | Answered count is informational only and never gates submission (server-side finalize remains authoritative); a tampered count cannot bypass any control |
</threat_model>
