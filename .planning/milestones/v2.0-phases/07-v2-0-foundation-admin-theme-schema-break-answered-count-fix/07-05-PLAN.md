<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| browser → dark-mode toggle | Client-only preference write to localStorage (no server trust) |
| lecturer form → section/lecturer-assignment POST | UI forms submit to the ownership-gated 07-04 endpoints; the view must not fabricate cross-subject actions |
| status string → status-pill class | Section window state mapped to a pill palette |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-07-14 | Elevation of Privilege | section/lecturer-assignment forms | mitigate | Forms post to the 07-04 routes whose Form Requests enforce per-subject ownership (authorize()); the view offers actions only for the subject in scope, and server-side authorize() is the real gate regardless of rendered affordances |
| T-07-15 | Information Disclosure (XSS) | status-pill + section/lecturer names | mitigate | Status flows through the fixed status-pill allowlist; all user-authored text (subject/section/lecturer names) rendered via escaped Blade, never `{!! !!}` |
| T-07-16 | Information Disclosure | dark-mode toggle state | accept | Client-only UI preference; no server round-trip, no sensitive data |
</threat_model>
