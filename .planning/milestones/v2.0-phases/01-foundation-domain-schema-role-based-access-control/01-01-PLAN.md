<pivot_naming_resolution>
The two pivot tables' names are resolved explicitly here (do not leave both floating — per RESEARCH Pitfall 5):

- **`classroom_subject`** (classroom ↔ subject): supersedes ARCHITECTURE.md / ROADMAP's illustrative `class_subject`. This IS Laravel's `belongsToMany` convention for the `Classroom` + `Subject` models (alphabetical singular), so relationship methods need **no** pivot override. Columns: `classroom_id`, `subject_id`.
- **`exam_classroom`** (exam ↔ classroom): kept exactly as ROADMAP success criterion #1 / ARCHITECTURE.md name it. Because Laravel's alphabetical convention would otherwise infer `classroom_exam`, the `belongsToMany` calls in plan 01-02 MUST pass the pivot table name `exam_classroom` explicitly. Columns: `exam_id`, `classroom_id`.

Every migration, model, factory, seeder, and test in this phase uses these two names consistently.
</pivot_naming_resolution>
